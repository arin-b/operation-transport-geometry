# Proposal-aligned implementation mapping

This repository now implements Operational Transport Geometry as a graph-level object, not as a two-domain one-node feature-alignment proxy.

| Proposal notation | Implementation |
|---|---|
| Deployed finite DAG `G=(V,E)` | `GraphWorldBatch.graph`, generated by `SyntheticDAGWorld` with nodes `input`, `detector`, `representation`, `measurement`, `audit_report` |
| Deployment domain set `D` | `GraphWorldBatch.domains`, a dictionary of `DomainSpec` objects |
| Input law `P_d` | `DomainSpec.input_law_metadata` and the synthetic domain sampler in `otg/worlds/synthetic_graph.py` |
| Node-wise law `mu_v^d=(Phi_v)#P_d` | `NodeLaw.samples`, keyed by `(node, domain)` in `GraphWorldBatch.node_laws` |
| Shared-space projection `phi_v^d` | `NodeLaw.projection_metadata`; identity and affine/noisy domain-specific projections are recorded |
| Aligned law `mutilde_v^d` | `NodeLaw.aligned_samples` |
| Terminal evaluation `E_op(Y)` | `GraphWorldBatch.terminal_evaluation_metadata`; terminal `audit_report` contains adequacy, coverage, glare/confounder, uncertainty |
| Node-wise operational risk `r_v(z)` | `NodeLaw.true_risk`, induced from terminal audit behavior, not assigned as an unrelated label |
| Estimated risk `rhat_v` | Pairwise risk estimators in `otg/risks/`, applied after deriving `PairwiseNodeProblem` |
| Operational cost `c_{v,op}` | `otg/costs/operational.py` |
| Admissible couplings `Pi_{v,adm}` | `InvarianceOutput.allowed` from `otg/invariance/` |
| Node discrepancy `W_{v,op}(mu_v^d,mu_v^{d'})` | `NodePairResult.transport.value` |
| System discrepancy `D_op(d,d')` | `SystemComparisonResult.aggregate.value` and `D_op_matrix.csv` |
| Unbalanced transport | `transport.solver=unbalanced`, with dangerous unmatched mass diagnostics |

The old pairwise arrays (`z_a`, `z_b`, etc.) are now an internal compatibility object named `PairwiseNodeProblem`. They are derived from graph-level node laws and no longer define the whole repository.
